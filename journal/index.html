Game Design UX Best Practices

User friendly game design can help guide the users to create an impactful user experience. One example of user-friendly game design is designing around the user’s “comfort to reach” zone. These easy to reach areas are near the bottom of the screen, close to the user's hands, where it is easy for the user to tap important control buttons. Utilizing sliders is another method of creating a user-friendly experience when all the content does not fit on one screen. To make sliders intuitive for a user, it is beneficial to make slider content partially visible at the edges of the screen. You can also animate the content of the slider from end to start when the user launches the content, so they are able to quickly see that there is more content they can explore. Pop-ups are another tool that can be utilized and designed for good game design. A good practice for user-initiated pop-ups is to animate their initiators to create association between the button and its pop-up. It is also beneficial to dim the screen behind the pop-up so the user is not distracted by background content. When the pop-up content contains valuable information the user should read, it is better to omit the “X” button in the top corner. Using a close button at the bottom will make the user more likely to read the content of the pop-up. It is also beneficial to organize user decision buttons by placing the button that is better for the game on the right side and the other option on the left, since users more intuitively select the right side.

Visual Thinking Strategies

Observing imagery is a great way to practice visual and critical thinking skills. Complex, intriguing imagery, with context that is not always clear, encourages the viewer to observe deeper and look for more details in order to make sense of the image. Often, deciphering a complex image is rewarding for the viewer because it allows them to think like a detective and create a meaning that makes sense to them. These kinds of “what’s going on?” images introduce viewers to new ideas, allowing them to try new observational approaches and encouraging them to pay attention to details.

The images are abstract and encourage the viewer to further explore what they are looking at by hovering over more locations, scrolling, and clicking.

Best Practices for Modal Design

Modals are a great tool for simplifying user interface and saving screen space. They are a useful tool for getting user information or displaying additional information. However, the overuse of modals has caused users to instinctively avoid modals by automatically dismissing them whenever they pop up. An important way to make modals user-friendly is by letting the users’ actions trigger the modal, as surprise modals may trigger the user to automatically close them. It is also important to always give the user a clear path to escape a modal. This can be done with exit buttons, escape options, cancel buttons, or letting the user click outside the modal. Another important part of modal design is giving the modal a descriptive title so the purpose of the modal is made clear. It is also important to size modals properly according to the size of the window. This article recommends that a modal should not be larger than 50% of the window space. It is also helpful to darken the background of the page to make the modal stand out. Modals also do not work well on mobile because of the small screen size.

Best Practices for Form Design

The main best practice for form design is keeping the form as simple as possible; "less is more." It is important that forms accommodate the user, since most people do not enjoy filling out forms. Creating an experience for the user that is as straight forward as possible will encourage users to complete form fields. Forms should have a simple layout that allow the user to view the entire form before completion. This can be done buy putting the form into a single column layout. It is also important to put the easier questions first and to clearly distinguish the required and optional fields from each other. Adding labels to and sizing input fields accordingly can also help guide the user.

https://uxdesign.cc/best-practices-for-form-design-ff5de6ca8e5f

One form I believe is user friendly is the Aritzia create an account form. The information is organized into a column from the most to least simple inputs, everything is labeled, and is is made clear which parts are optional and which are required.
